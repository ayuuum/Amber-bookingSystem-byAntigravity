# 複数エディタでの開発ワークフロー

## Cursor と Antigravity の併用について

### ✅ 基本的には問題ありません

Cursor と Antigravity の両方から同じプロジェクトを編集することは**技術的に可能**です。ただし、以下の点に注意してください。

---

## ⚠️ 注意点

### 1. ファイルの競合

**問題**: 両方のエディタで同じファイルを同時に編集すると、保存時に一方の変更が上書きされる可能性があります。

**対策**:
- 同じファイルを同時に編集しない
- ファイルを保存する前に、もう一方のエディタで変更がないか確認
- Git を使用して変更を管理（推奨）

### 2. エディタ固有の設定ファイル

**現在の状況**:
- `.cursor/` ディレクトリが存在（MCP設定など）
- `.vscode/` ディレクトリは存在しない
- `.antigravity/` ディレクトリは存在しない

**推奨事項**:
- エディタ固有の設定ファイルは `.gitignore` に追加することを検討
- プロジェクト全体で共有する設定（例: `tsconfig.json`, `eslint.config.mjs`）は両方のエディタで使用可能

### 3. Language Server (LSP) の競合

**問題**: TypeScript や ESLint の Language Server が両方のエディタで起動すると、リソースを消費する可能性があります。

**対策**:
- 通常は問題ありませんが、パフォーマンスが気になる場合は、使用していないエディタを閉じる
- 各エディタの Language Server 設定を確認

### 4. ファイルウォッチャーの競合

**問題**: Next.js の開発サーバー（`npm run dev`）はファイル変更を監視します。両方のエディタから保存すると、ホットリロードが複数回発生する可能性があります。

**対策**:
- 通常は問題ありませんが、予期しない動作がある場合は、一方のエディタを閉じて確認

### 5. キャッシュとインデックス

**問題**: 各エディタは独自のキャッシュやインデックスを保持します。一方のエディタで作成したファイルが、もう一方のエディタで即座に認識されない場合があります。

**対策**:
- ファイルを保存した後、もう一方のエディタでファイルを開き直す
- エディタの「ファイルの再読み込み」機能を使用

## ✅ 推奨ワークフロー

### パターン1: 機能別に分ける（推奨）

```
Cursor: フロントエンド開発（React/Next.js）
Antigravity: バックエンド開発（API、データベース）
```

### パターン2: 時間帯で分ける

```
午前: Cursor で作業
午後: Antigravity で作業
```

### パターン3: タスク別に分ける

```
Cursor: 新機能開発
Antigravity: バグ修正・リファクタリング
```

## 🔧 設定の推奨事項

### .gitignore への追加（オプション）

エディタ固有の設定ファイルを Git に含めない場合は、以下を `.gitignore` に追加：

```gitignore
# Editor directories
.vscode/
.cursor/
.antigravity/
*.code-workspace
```

**注意**: `.cursor/mcp.json` などの重要な設定ファイルは、チームで共有する場合は Git に含めることを検討してください。

### 共有設定ファイル

以下のファイルは両方のエディタで使用可能です：
- `tsconfig.json` - TypeScript 設定
- `eslint.config.mjs` - ESLint 設定
- `components.json` - shadcn/ui 設定
- `next.config.ts` - Next.js 設定

## 🚨 トラブルシューティング

### 問題: ファイルが保存されない

**原因**: もう一方のエディタでファイルが開かれている可能性

**解決策**:
1. 両方のエディタでファイルを閉じる
2. 再度開いて保存を試す
3. それでも解決しない場合は、エディタを再起動

### 問題: 型エラーが表示されない

**原因**: Language Server が正しく起動していない可能性

**解決策**:
1. エディタを再起動
2. TypeScript サーバーを再起動（コマンドパレットから）
3. `node_modules` を削除して `npm install` を再実行

### 問題: ホットリロードが動作しない

**原因**: 開発サーバーが複数のファイル変更を正しく検知できていない可能性

**解決策**:
1. 開発サーバーを再起動
2. `.next` ディレクトリを削除して再起動

## 📝 ベストプラクティス

1. **Git を活用**: 変更をこまめにコミットして、競合を防ぐ
2. **ブランチ戦略**: 機能ごとにブランチを作成して作業
3. **コミュニケーション**: チームで作業する場合は、どのエディタを使用しているか共有
4. **定期的な同期**: 両方のエディタで定期的にファイルを再読み込み

---

## 結論

**Cursor と Antigravity の併用は問題ありません**が、上記の注意点を守ることで、スムーズに開発を進められます。

特に重要なのは：
- ✅ 同じファイルを同時に編集しない
- ✅ Git で変更を管理する
- ✅ 定期的にファイルを再読み込みする
