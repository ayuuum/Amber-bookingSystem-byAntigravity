# Amber Design Guidelines（日本市場最適化版）

> [!NOTE]
> 本ドキュメントは [PRD](file:///Users/ayumu/Amber-House-PJ/docs/prd.md) に掲げられたプロダクトビジョンと課題解決を、ビジュアルおよび体験として具現化するための指針である。

## 1. デザインコンセプト： "Japan-Modern Trust"

日本国内の現場で愛され、老若男女問わず迷わず使える **「清潔感」「高い視認性」「誠実さ」** をデザインの核とする。グローバルな先進性をスパイスとしつつ、日本人が直感的に「使いやすい」と感じる UI/UX を追求する。

### 参考にすべき日本国内の SaaS
- **[SmartHR](https://smarthr.jp/)**: 日本で最も成功している SaaS デザイン。清潔感のある配色、親しみやすい言葉選び、高いアクセシビリティが特徴。
- **[Sansan](https://jp.sansan.com/)**: ビジネスとしての「硬さ」と「洗練」のバランスが絶妙。プロフェッショナルな信頼感を醸成するデザイン。
- **[マネーフォワードクラウド](https://biz.moneyforward.com/)**: 複雑な情報を整理して見せるグリッドシステムと、安心感を与える色の使い方が参考になる。

## 3. タイポグラフィ（Typography）

日本語の読みやすさを科学する。

- **フォントファミリー**: `IBM Plex Sans JP` または `Noto Sans JP` を最優先。
  - OS 標準フォント (`Hiragino Kaku Gothic ProN`, `Meiryo`) へのフォールバックを完璧に行う。
- **行間 (Line Height)**: 本文は `1.6` 〜 `1.8` を確保。日本語特有の詰まり感を解消する。
- **階層構造**:
    - 日本人は「タイトル + 説明文」の構成に慣れているため、見出しだけでなく補足説明（Microcopy）を積極的に配置する。

## 4. UI 構成の原則（UI Principles）

### 4-1. 情報のグループ化（Card Layout）
- 要素を枠線（Border）や背景色で明確に囲む。日本人は「どこからどこまでが一つの塊か」がはっきりしているデザインを好む。
- 角丸（Rounding）は `8px` (md) を標準とし、丸すぎず硬すぎない「誠実さ」を表現する。

### 4-2. ボタンのデザイン
- **Primary**: Amber色（`#F59E0B`）。はっきりと「次に押すべきこと」を伝える。
- **ラベル**: 「Submit」ではなく「予約を確定する」など、動詞を含む具体的な日本語を使用する。

### 4-3. 安心感を与えるフィードバック
- 操作に対して必ず「完了しました」「保存しました」といったトースト通知を表示する。
- フォームのバリデーションは、エラーが起きてから出すのではなく、入力中にリアルタイムで「OK」を出す、親切な設計（SmartHR方式）を目指す。

---

## 5. モーション ＆ グラフィック

- **派手さよりスムーズさ**: 複雑なアニメーションは避け、0.2秒程度の短いフェードやスライドで、操作の「手応え」を伝える。
- **アイコンとテキストの併記**: アイコン単体での表示は避け、必ず日本語テキストを添える。これにより、IT に不慣れなスタッフや顧客の迷いをなくす。

---

## 6. 日本市場特有の検討事項

- **住所・電話番号の形式**: 日本独自の入力形式（ハイフン、郵便番号検索）をデフォルトで組み込む。
- **全角・半角の自動変換**: ユーザーがどちらで入力してもシステム側で適切に処理する（ユーザーに修正させない「おもてなし」設計）。
- **LINE 優先の UX**: 日本人のチャネルはメールではなく LINE である。LINE 連携を前提とした UI パーツ（リッチメニュー、LIFF）を常に意識する。
